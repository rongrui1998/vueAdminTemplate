# V2 标准版菜单管理设计

## 背景

`front/V2` 已经有了 `系统管理 -> 菜单管理` 入口，也已经打通了菜单数据的 mock/API 读取能力，但当前页面仍然停留在“说明卡片 + 平铺记录”的占位状态，离标准后台菜单管理页还有明显距离。

这次目标不是继续补说明信息，而是把它收成真正可操作的标准页：

- 用树形表格承载目录 / 菜单 / 按钮结构
- 支持新增菜单、修改、新增下级、删除
- 同一套页面支持 `mock / api` 切换
- 为后续角色权限、账号权限验证提供真实配置入口

## 目标

- 页面样式接近标准暗色后台菜单管理页
- 菜单列表从卡片视图改为树形表格
- 支持目录 / 菜单 / 按钮三种类型的创建与编辑
- mock 模式下先完整闭环，后端接口结构与前端保持一致
- 交互和数据模型足够支撑下一步角色菜单授权

## 本次纳入范围

### 页面能力

- 顶部工具栏
- 树形表格展示
- 刷新数据
- 展开 / 收起全部
- 新增菜单
- 新增下级
- 修改菜单
- 删除菜单

### 表单能力

- 目录 / 菜单 / 按钮类型切换
- 上级菜单选择
- 基础字段录入
- 按类型动态控制字段显隐
- 表单校验

### 数据能力

- `GET /system/menus`
- `POST /system/menus`
- `PUT /system/menus/:id`
- `DELETE /system/menus/:id`
- mock 与 backend 采用一致的数据结构

## 本次暂不纳入

- 拖拽排序
- 批量导入导出
- 多条件筛选
- 组件路径实时远程校验
- 更复杂的菜单元信息，例如外链、iframe、按钮分组生成器

## 页面设计

### 1. 顶部工具栏

顶部右侧采用“主按钮 + 轻图标按钮”结构：

- `新增菜单`
- `刷新`
- `展开 / 收起`
- 预留一个轻操作位，用于后续密度或列设置

视觉上延续当前暗色主题，但弱化说明型卡片，增强管理台密度，让页面一打开就是“可操作”而不是“可阅读”。

### 2. 树形表格

表格列固定为：

- 标题
- 类型
- 权限标识
- 路由地址
- 页面组件
- 操作

标题列负责承载层级缩进、菜单图标、状态角标。目录、菜单、按钮在展示上遵循不同优先级：

- 目录：突出结构层级
- 菜单：突出路由与组件
- 按钮：突出权限标识

操作列提供：

- `新增下级`
- `修改`
- `删除`

其中按钮类型默认不再新增子级。

### 3. 弹窗表单

新增、修改、新增下级共用一个 `MenuFormDialog`。

核心字段：

- 上级菜单
- 类型
- 标题
- 路由地址
- 页面组件
- 权限标识
- 图标
- 排序
- 状态
- 是否缓存
- 是否隐藏
- 备注

字段显隐规则：

- `directory`：保留标题、路径、图标、排序、状态、隐藏等结构字段
- `menu`：显示路由、组件、权限、缓存等完整字段
- `button`：仅保留标题、权限、排序、状态、备注等轻字段

### 4. 删除确认

删除采用二次确认，文案明确提示：

- 删除目录时会影响子级菜单
- 删除按钮时只影响权限节点

若当前节点存在子级，mock 和后端都应按统一规则处理。首版建议直接允许递归删除，并在确认文案中说明。

## 前端实现设计

### 1. API 层

`src/api/system-menu.ts` 从只有 `getSystemMenusApi` 扩成完整 CRUD：

- `getSystemMenusApi`
- `createSystemMenuApi`
- `updateSystemMenuApi`
- `deleteSystemMenuApi`

页面不关心当前是 mock 还是 api，仍通过统一 request 层发起请求。

### 2. 视图拆分

建议拆成：

- `src/views/system/menu/index.vue`
- `src/views/system/menu/components/MenuFormDialog.vue`
- `src/views/system/menu/constants.ts`

`index.vue` 负责：

- 工具栏
- 表格展示
- 弹窗开关
- 删除确认

`MenuFormDialog.vue` 负责：

- 表单渲染
- 动态字段显隐
- 校验
- 提交事件抛出

### 3. mock 与 backend 对齐

mock 模块先行实现：

- 列表查询
- 新增
- 编辑
- 删除

后端接口保持一致的 URL 和 payload 结构，这样页面切 `api` 时无需改 UI 逻辑。

## 错误处理

- 首次加载失败：表格区显示错误态和重试按钮
- 提交失败：弹窗内保留表单内容，显示错误提示
- 删除失败：保留当前列表状态并给出提示

## 测试设计

至少覆盖：

- 列表加载成功
- 展开后的树结构可见
- 点击 `新增菜单` 可以打开弹窗
- 菜单类型切换后字段显隐正确
- 新增菜单成功后表格刷新
- 修改菜单成功后数据更新
- 删除菜单成功后节点消失

## 验收标准

- 菜单管理页不再出现说明型大卡片主视图
- 页面主视图为树形表格
- mock 模式下完成新增、编辑、删除闭环
- 表单字段显隐符合目录 / 菜单 / 按钮语义
- 后续切换 `api` 模式时无需重写页面结构
