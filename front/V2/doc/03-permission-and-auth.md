# 登录、权限与页面级控制

## 登录流程

1. 登录页提交账号密码
2. 登录成功后保存 token
3. 访问受保护页面时，在守卫中拉取用户信息
4. 再拉菜单并生成动态路由
5. 最后放行页面

关键文件：

- `src/views/login/index.vue`
- `src/store/modules/auth.ts`
- `src/router/guards.ts`
- `src/utils/auth.ts`

## Base 权限范围

Base 版本当前已完成：

- 登录态校验
- 菜单级控制
- 页面级权限兜底
- 按钮级权限控制
- 权限组合式 API

补充说明：

- 左侧菜单只展示当前账号可见的页面
- 某些无权限页面会以隐藏路由形式保留，用于直达地址时正确进入 `403`
- 所以“菜单里看不见”和“手动访问后被拦截”是两层不同能力

不做：

- RBAC 权限矩阵
- 数据权限

## 页面级权限兜底

动态菜单本身已经控制了大部分页面访问范围。

为了避免菜单数据异常或手工访问路由时缺少显式权限判断，Base 额外增加了 `route.meta.permission` 校验：

- 菜单生成路由时会把 `permission` 写入 `meta`
- 路由守卫在放行前校验当前用户是否具备该 permission
- `admin` 角色默认放行
- 若无权限，Base 版本跳转到独立 `403` 页面
- 如果目标页面被隐藏但路由已注入，手动访问时同样会进入 `403`，不会误落到 `404`

## 按钮级权限

当前支持两种使用方式：

- 指令：`v-permission="'demo:crud:create'"`
- 组合式：`usePermission()`

适用范围：

- 操作按钮
- 局部工具栏
- 需要根据权限做显隐或禁用的业务区块

## 用户信息字段

当前用户信息包含：

- `roles`
- `permissions`

mock 用户在本地开发环境中会返回基础权限数据，便于验证页面级控制。
