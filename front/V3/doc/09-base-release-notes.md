# V3 Base 定版说明

## 1. 定版结论

`front/V3` 当前可以正式作为 **Base 版本** 定版使用。

这意味着它已经具备：

- 可启动
- 可演示
- 可复用
- 可说明
- 可验证

当前版本适合作为团队后台项目的统一起步模板，也适合作为后续 Standard 版本演进的基础分支。

## 2. 本次定版覆盖范围

### 已完成能力

- 登录、退出登录、401 统一处理
- 用户信息拉取与本地持久化
- 动态菜单、动态路由、动态路由恢复
- 页面级权限、按钮级权限、权限组合式 API
- `403 / 404 / route-missing` 错误页
- 左侧菜单、顶部栏、面包屑、标签页
- 左侧和顶部固定，中间内容区独立滚动
- 明暗主题、全屏切换、消息通知面板
- CRUD 示例页
- 权限示例页
- 多级菜单示例页
- mock 登录、mock 菜单、mock CRUD
- 基础测试、提交前校验、CI
- Base 总览文档、使用文档、后端对接文档

### 已收口问题

- 权限示例页动态路由 id 冲突导致的 `404`
- 无权限隐藏页面直达地址误落 `404`，现已改为正确进入 `403`
- 顶部消息、全屏、标签页等交互的基础测试覆盖
- Base 文档入口分散的问题

## 3. 当前质量门槛

定版以以下验证为准：

```bash
pnpm lint
pnpm stylelint
pnpm test
pnpm build
```

当前这些命令通过后，可认为 Base 主链路处于可交付状态。

## 5. 当前版本边界

当前定版是 **Base**，不是 **Standard**。

因此仍然明确不包含：

- 完整 RBAC 管理后台
- 用户/角色/菜单维护页面
- 数据权限体系
- 字典体系
- 上传、导入导出、复杂表单等更重型通用业务组件

这些能力建议放到下一阶段推进。

## 6. 剩余非阻塞事项

当前仍有少量非阻塞收尾项：

- Dart Sass 存在 `legacy-js-api` 的上游兼容提示
- `404` 页面视觉层级还可以继续与 `403` 统一
- 可以继续补更多组件级测试，但不影响 Base 定版

这些不影响当前作为 Base 模板交付。

## 7. 建议的后续动作

如果你准备把这版作为一个明确节点使用，建议：

1. 以当前状态打一个 `V3 Base` 版本标签或里程碑
2. 后续新增需求按 `Standard` 阶段单独规划
3. 保持 `doc/` 目录作为唯一文档入口，不再混用旧路径
