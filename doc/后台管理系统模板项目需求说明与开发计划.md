# 后台管理系统模板项目需求说明与开发计划

## 一、项目背景

为提升团队后台项目的开发效率，减少重复搭建基础工程、布局框架、权限逻辑和通用页面的成本，计划建设一套统一的前端后台管理系统模板项目。该模板项目应具备良好的可复用性、可扩展性和可维护性，既能够满足基础业务项目快速启动，也能够支持后续逐步扩展为标准化中后台解决方案。

本项目定位不是单一演示系统，而是一套可供团队后续业务项目直接复用的前端模板基础工程。

---

## 二、项目目标

### 1. 总体目标

建设一套基于 Vue 3 技术栈的后台管理系统模板，形成统一的：

- 工程化基础能力
- 页面布局能力
- 登录与权限接入能力
- 菜单渲染能力
- 通用 CRUD 页面模板能力
- 常用后台组件复用能力

### 2. 建设目标

通过分阶段建设方式，逐步形成三个层级版本：

- **Base 基础版**：满足后台项目快速启动
- **Standard 标准版**：满足通用中后台系统建设
- **Pro 高级版**：沉淀高复用后台组件与增强能力

---

## 三、技术栈说明

### 1. 核心技术栈

- **Vue 3**
- **TypeScript**
- **Vite**
- **Pinia**
- **Vue Router**
- **Element Plus**

### 2. 配套能力建议

- **Axios**：请求封装
- **Sass / SCSS**：样式管理
- **ESLint + Prettier**：代码规范
- **Mock 能力**：支持前后端未联通时独立开发与演示
- **pnpm**：统一包管理

### 3. UI 方案说明

本项目初版采用 **Element Plus** 作为 UI 组件库，原因如下：

- Vue 生态后台场景成熟
- 表格、表单、弹窗、树形、分页等能力完善
- 团队上手快，学习成本低
- 社区资料丰富，适合作为标准模板基础

说明：项目初期不考虑支持多 UI 框架切换，避免过度设计。

---

## 四、核心需求说明

### 1. 基础工程能力

模板项目需具备以下基础工程能力：

- 基于 Vite 的工程初始化
- TypeScript 类型支持
- 路由管理与页面组织
- Pinia 状态管理
- 环境变量管理
- 统一请求封装
- 全局样式与主题变量管理
- ESLint / Prettier 规范能力
- Mock 数据支持

### 2. 登录与认证能力

模板项目需支持基础登录流程，包括：

- 登录页面
- 账号密码登录
- 登录表单校验
- token 存储与读取
- 登录态校验
- 路由守卫拦截
- 退出登录
- 登录失效统一处理（如 401）

说明：首期采用常规后台项目通用登录方案，以 **token + 用户信息** 为主，refresh token 机制暂不作为首版必做项，但需预留扩展空间。

### 3. 菜单与路由能力

项目已明确采用 **后端返回菜单，前端动态渲染** 的方式。

#### 菜单能力要求

- 登录后获取菜单数据
- 根据后端返回菜单生成左侧菜单
- 支持多级菜单
- 支持菜单图标
- 支持菜单排序
- 支持隐藏菜单
- 支持动态路由映射
- 支持刷新后路由恢复
- 支持菜单高亮联动

#### 路由能力要求

- 静态基础路由
- 动态业务路由
- 登录拦截
- 404 页面
- 页面缓存控制（keepAlive）

说明：Base 版本即接入菜单控制能力，不采用纯静态菜单方案，以减少后续 Standard 阶段的重构成本。

### 4. 菜单与权限字段约定

菜单和权限字段采用常规后台管理系统登录后返回菜单信息的方式，接口响应结构参考以下形式：

```json
{
  "code": 200,
  "data": {
    "total": 10,
    "list": [
      {
        "id": "69c78fbb475b11bbe1a7c822",
        "name": "菜单名称"
      }
    ]
  },
  "msg": "success",
  "time": "2026-04-25T09:11:48.666Z",
  "tip": "成功"
}
```

建议菜单数据至少包含以下字段：

- `id`
- `parentId`
- `name`
- `path`
- `component`
- `icon`
- `sort`
- `type`（目录 / 菜单 / 按钮）
- `permission`（权限标识）
- `hidden`
- `keepAlive`
- `affix`
- `status`
- `children`

说明：Base 版本先实现菜单级和页面级控制；Standard 版本再补充按钮级权限控制。

### 5. 权限能力

权限能力按阶段建设：

#### Base

- 登录态校验
- 菜单级访问控制
- 页面路由控制

#### Standard

- 引入 **RBAC（基于角色的权限控制）**
- 菜单权限
- 按钮权限
- 页面权限
- 权限指令封装
- 动态权限过滤

#### Pro

- 在 Standard 基础上预留更复杂权限扩展能力，如数据权限扩展点

说明：当前明确采用后台常规 RBAC 方案，不在首版引入复杂数据权限模型。

### 6. 布局与导航能力

模板项目需具备标准后台布局能力：

- 左侧菜单栏
- 顶部导航栏
- 面包屑导航
- 主内容区
- 响应式菜单折叠
- 标签页导航（多页签）

#### 标签页要求

- 首页固定不可关闭
- 支持关闭当前
- 支持关闭其他
- 支持关闭全部
- 支持标签页与路由联动
- 支持标签页刷新后恢复
- 支持与 keepAlive 配合使用

### 7. 主题与视觉模式

项目已确认：

- **Base 不做完整主题切换系统**
- **Base 需要支持暗黑模式**
- 主题能力先控制在 light / dark 两种模式

#### 暗黑模式要求

- 支持明亮模式 / 暗黑模式切换
- 支持用户本地记忆选择结果
- 支持 Element Plus 暗黑样式适配
- 支持基础自定义样式同步适配

说明：首期仅实现暗黑模式，不扩展主题色、多套皮肤等完整主题系统，避免范围过大。

### 8. 通用请求封装

项目需提供统一请求层，支持：

- axios 实例封装
- request 拦截器
- response 拦截器
- token 自动注入
- 错误码统一处理
- 统一错误提示
- 文件下载封装
- 接口模块化组织

#### 接口响应结构

接口返回结构参考常规后台风格，例如：

- `code`
- `msg`
- `data`

列表数据通常包含：

- `total`
- `list`

说明：请求层需兼容常规后台接口格式，便于后续快速接入真实后端接口。

### 9. Mock 能力

项目需提供 mock 支撑能力，满足模板独立开发、调试和演示。

#### 建议 mock 范围

- 登录接口
- 获取用户信息接口
- 获取菜单接口
- Dashboard 数据接口
- 用户列表接口
- 角色列表接口
- 菜单列表接口
- 通用增删改查示例接口

说明：Mock 不是辅助项，而是模板首期的重要组成部分，用于确保项目在未联通真实后端时也可完整展示。

### 10. 通用业务页面模板能力

模板需提供可直接复用的基础业务页面模板，至少包括：

- 表格查询页
- 弹窗表单页
- 抽屉详情页
- 分页能力
- 搜索 / 重置能力
- 状态切换
- 删除确认
- 加载态
- 空状态
- 异常状态

目标是让业务开发可以基于模板页面快速复制和改造，而不是只提供空壳结构。

### 11. 交付物要求

模板项目交付物不仅包含模板源码，还应包含以下内容：

- 模板项目源码
- 项目目录结构说明
- 菜单接入说明
- 权限接入说明
- Mock 使用说明
- 通用页面开发规范
- 组件复用说明
- 后续扩展建议

## 五、版本规划

### 1. Base 基础版

#### 目标

快速形成一套可直接启动业务开发的后台模板基础工程。

#### 功能范围

- 登录页
- token 登录态管理
- 路由守卫
- 基础布局
- 左侧菜单
- 顶部导航
- 面包屑
- 标签页
- 后端返回菜单的动态渲染
- 页面级菜单控制
- 首页 Dashboard
- 暗黑模式
- 请求封装
- Mock 能力
- 基础 CRUD 页面模板
- 404 页面

#### 版本定位

适合快速起一个基础后台项目，重点解决“从 0 到 1”的启动效率问题。

### 2. Standard 标准版

#### 目标

形成具备通用系统管理能力的标准中后台模板。

#### 在 Base 基础上新增

- RBAC 权限模型
- 按钮级权限控制
- 用户管理
- 角色管理
- 菜单管理
- 权限指令
- 动态权限过滤
- 异常页（401 / 403 / 404）
- 字典能力

#### 版本定位

适合作为中小型后台系统的标准起步模板，是本项目最核心的可落地版本。

### 3. Pro 高级版

#### 目标

沉淀高复用后台组件与增强能力，提升模板的工程化深度和页面开发效率。

#### 在 Standard 基础上新增

- `ProTable`
- `SearchForm`
- `ModalForm`
- `DrawerForm`
- `PageContainer`
- 状态标签组件
- 字典渲染组件
- 上传组件封装
- 导入导出能力
- 多布局模式
- 更完整的全局设置项
- 更完整的文档与示例页

#### 版本定位

作为高级后台模板能力沉淀，面向更复杂、更长期复用的后台项目。

说明：Pro 版本参考 vben-admin 的设计思路，但不追求完全对标其复杂度，重点是“够用、易扩展、团队易理解”。

## 六、开发计划

### 阶段一：项目基础骨架建设

#### 目标

完成模板项目的底层工程初始化。

#### 主要内容

- 初始化 Vue 3 + TypeScript + Vite 项目
- 集成 Element Plus
- 集成 Vue Router / Pinia
- 建立目录结构
- 配置请求层
- 配置样式体系
- 配置 ESLint / Prettier
- 配置 Mock 机制

#### 交付结果

形成可运行的基础工程骨架。

### 阶段二：Base 基础版功能建设

#### 目标

完成后台基础模板能力，支持快速启动业务项目。

#### 主要内容

- 登录页开发
- 登录态管理
- 动态菜单渲染
- 基础布局开发
- 顶部导航 / 面包屑 / 左侧菜单
- 标签页能力
- Dashboard 页面
- 暗黑模式
- 通用 CRUD 示例页面
- 404 页面
- Mock 接口联调

#### 交付结果

形成可演示、可复用、可直接起业务页面的基础版模板。

### 阶段三：Standard 标准版功能建设

#### 目标

补齐标准中后台系统所需的权限和系统管理模块。

#### 主要内容

- RBAC 权限设计与实现
- 用户管理页
- 角色管理页
- 菜单管理页
- 按钮权限控制
- 权限指令封装
- 异常页补齐
- 字典能力接入

#### 交付结果

形成标准中后台模板，满足多数后台业务项目启动需求。

### 阶段四：Pro 高级版能力沉淀

#### 目标

沉淀高复用业务组件和增强能力。

#### 主要内容

- ProTable 封装
- SearchForm 封装
- ModalForm / DrawerForm 封装
- 上传与导入导出能力
- 多布局与设置项
- 页面容器与状态组件
- 文档示例完善

#### 交付结果

形成面向复杂后台项目的高级模板能力库。

## 八、实施建议

1. **优先完成 Base**
   - 先形成能用的模板骨架
   - 尽快具备演示和复用能力

2. **再推进 Standard**
   - 补齐系统管理和权限体系
   - 将模板提升到可直接支撑业务项目的程度

3. **Pro 作为增强项**
   - 根据时间和实际业务复用价值推进
   - 不建议一开始就做过重

---

## 九、结论

本项目建议采用“**分阶段建设、逐步沉淀**”的方式推进：

- **Base** 解决后台项目快速启动问题
- **Standard** 解决标准中后台系统建设问题
- **Pro** 解决高复用组件和高级能力沉淀问题

该方案兼顾交付速度、落地价值和后续扩展空间，适合作为团队统一后台模板项目的建设方向。
